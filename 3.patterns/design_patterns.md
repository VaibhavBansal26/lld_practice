# Design Patterns

Main categories of design patterns:
1. **Creational Patterns:** Deal with object creation mechanisms, trying to create objects in a manner suitable to the situation. Examples include Singleton, Factory Method, Abstract Factory, Builder, and Prototype patterns.
2. **Structural Patterns:** Concerned with how classes and objects are composed to form larger structures. Examples include Adapter, Bridge, Composite, Decorator, Facade, Flyweight, and Proxy
3. **Behavioral Patterns:** Focus on communication between objects, what goes on between objects and how they operate together. Examples include Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, and Visitor patterns.
4. **Concurrency Patterns:** Address issues related to multi-threading and concurrent execution. Examples include Active Object, Balking, Double-Checked Locking, Guarded Suspension, Monitor Object, Scheduler, Thread Pool, and Read-Write Lock patterns.
5. **Architectural Patterns:** Provide high-level structures for software systems. Examples include Model-View-Controller (MVC), Model-View-ViewModel (MVVM), Microservices, Layered Architecture, Event-Driven Architecture, and Service-Oriented Architecture (SOA) patterns.
